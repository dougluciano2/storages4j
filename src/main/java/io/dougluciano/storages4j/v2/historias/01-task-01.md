# Task 01 — Criar a classe StorageConfiguration

## 📦 Pacote
io.dougluciano.storages4j.config

---

## 📛 Classe
StorageConfiguration

---

## 🎯 Objetivo
Centralizar todas as configurações necessárias para o funcionamento da biblioteca, sem acoplamento com fontes externas como `.properties`, variáveis de ambiente ou frameworks.

A classe deve ser apenas um objeto de configuração (POJO), sem lógica de execução.

---

## 🧩 Atributos

- StorageProvider provider
- String bucket
- String accessKey
- String secretKey
- boolean enableMetadata
- boolean useUniquePath

---

## 🛠️ Métodos (Builder fluente recomendado)

- provider(StorageProvider provider)
- bucket(String bucket)
- accessKey(String accessKey)
- secretKey(String secretKey)
- enableMetadata(boolean enableMetadata)
- useUniquePath(boolean useUniquePath)
- build()

---

## 💡 Exemplo de uso

StorageConfiguration config = StorageConfiguration.builder()
.provider(StorageProvider.MINIO)
.bucket("uploads")
.accessKey("user")
.secretKey("password")
.enableMetadata(true)
.useUniquePath(true)
.build();

---

## ⚠️ Regras importantes

- Não deve ler `.properties`
- Não deve conter lógica de negócio
- Não deve depender de frameworks
- Deve ser imutável após o build
- Deve validar campos obrigatórios no `build()`

---

## 🎯 Resultado esperado

Uma classe simples, clara e desacoplada, responsável apenas por representar a configuração da biblioteca, permitindo flexibilidade total para quem consome a API.
